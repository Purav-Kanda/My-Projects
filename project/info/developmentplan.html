Development Plan for Kevin Yang’s Portfolio Hub

1. Purpose

The purpose of the app is to create a professional, interactive, fun, and promotional platform showcasing Kevin Yang's skills, interests, hobbies, projects, and achievements. The app will serve as a living portfolio that Kevin can maintain throughout his university life and beyond, useful for future employers, colleagues, and others interested in learning more about Kevin.

2. Major Features

  • Instagram Integration: Display photos from Kevin's Instagram feed.
  • Resume Download: Provide a downloadable PDF of Kevin’s resume.
  • Contact Form: Offer a straightforward way for people to reach Kevin.
  • Project Showcase: Link Kevin’s projects in an engaging and interactive way, including games he created.
  • Quiz Section: A fun quiz about Kevin, managed via the admin panel.
  • GitHub Integration: Showcase Kevin's coding projects from GitHub.
  • Admin Dashboard: Allows Kevin to manage projects, view messages, and update quizzes.
  • Expandable Design: Allow Kevin to easily add more projects, achievements, and updates over time.

3. Data Description

The app will use a database to store and manage information.

  • Projects Table: (id INT PRIMARY KEY, title VARCHAR(255), description TEXT, github_link VARCHAR(255), image_path VARCHAR(255)).
  • Contact Messages Table: (id INT PRIMARY KEY, name VARCHAR(100), email VARCHAR(100), message TEXT, timestamp DATETIME).
  • Quiz Questions Table: (id INT PRIMARY KEY, question TEXT, option_a VARCHAR(255), option_b VARCHAR(255), option_c VARCHAR(255), option_d VARCHAR(255), correct_answer CHAR(1)).

4. Functions Performed by the App

  • View Kevin's Instagram pictures.
  • Download Kevin's resume.
  • Contact Kevin via an interactive form.
  • Explore and interact with showcased projects.
  • Take a fun quiz about Kevin.
  • View Kevin's GitHub projects.
  • Admin functionality to manage projects, messages, and quizzes.

5. PHP and JavaScript Role

  • PHP: Handles back-end logic, form submissions, data retrieval, and admin panel operations.
  • JavaScript: Provides interactivity, handles dynamic updates, and uses AJAX for asynchronous operations.

6. Team Member Roles

  • Purav Kanda: Back-end Development - Manages PHP logic, database integration, and admin dashboard (contact section and admin interface).
  • Dhyan Patel: Front-end Development - Works on UI/UX, interactive components, and admin functionality for the contact form.
  • Sahil Saraf: Works on the quiz section (JavaScript/PHP functionality).
  • Vedant Patel: Works on the projects section (JavaScript/PHP functionality).

7. AJAX Implementation

AJAX will be used in the following ways:

  • Real-time form validation for the contact form to improve user experience.
  • Asynchronous quiz updates to allow users to submit answers without refreshing the page.
  • Dynamically loading GitHub projects without reloading the page.
  • Admin dashboard enhancements, allowing updates to projects and quizzes without requiring full page reloads.

8. Incremental Delivery

  • Week 1 (April 4th): Develop static pages and basic features, including web pages with design elements and basic links (Entire Team).
  • Week 2: Set up database, contact form, and back-end logic (Purav, Dhyan).
  • Week 3: Implement interactive features (mini-games, quiz, project showcase) (Purav, Dhyan).
  • Week 4: Test, debug, and deploy final version (Entire Team).

9. Client Feedback

Kevin liked the design and overall structure. However, he requested an admin panel to manage projects, messages, and quizzes. This feature has been added to our plan. We will show him each increment for further feedback.
